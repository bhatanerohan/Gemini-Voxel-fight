Anticipated Questions

Q&A

How do you prevent dangerous generated code?

Weapon code runs in a sandboxed context object. A validator scans for dangerous patterns (eval, fetch, DOM access). Code only gets the Weapon SDK helpers â€” nothing else.

What if Gemini returns bad/broken code?

Multi-model fallback chain. Validation catches syntax errors. If all fails, a default fallback weapon equips. Player never gets stuck.

Doesn't API latency ruin gameplay?

All Gemini calls happen during natural pauses: wave rest periods, game over screen, forge UI. Never blocks real-time combat. Mid-combat quips are fire-and-forget.

Can players generate overpowered weapons?

The system prompt constrains weapon power budgets. The SDK normalizes damage values. The Arena God can counter with resistant enemies and arena changes.

How is this different from asset generation?

We don't generate images or static content. Gemini generates executable behavior, strategic decisions, and narrative. The AI is a living participant, not a content pipeline.

What's the API cost per session?

~1 batched call per wave + 1 weapon forge + 1 chronicle = roughly 10-15 API calls for a full 10-wave run. Efficient batching keeps it minimal.
